# ApparenceKit Environment Configuration

> Source: https://apparencekit.dev/docs/start/overview/

## Environment Variables

All configuration is passed via `--dart-define` at build/run time. No `.env` files are read at runtime.

### Required Variables

| Variable | Description |
|----------|-------------|
| `ENV` | Environment: `dev` or `prod` |
| `BACKEND_URL` | Supabase project URL |
| `SUPABASE_TOKEN` | Supabase anon key |

### Optional Variables

| Variable | Description |
|----------|-------------|
| `RC_ANDROID_API_KEY` | RevenueCat Android API key |
| `RC_IOS_API_KEY` | RevenueCat iOS API key |
| `MIXPANEL_TOKEN` | Mixpanel analytics token |
| `SENTRY_DSN` | Sentry error reporting DSN |
| `APP_STORE_ID` | Apple App Store ID (for rating prompts) |

## VSCode Configuration

Edit `.vscode/launch.json`:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Dev",
      "request": "launch",
      "type": "dart",
      "args": [
        "--dart-define=ENV=dev",
        "--dart-define=BACKEND_URL=YOUR_SUPABASE_PROJECT_URL",
        "--dart-define=SUPABASE_TOKEN=YOUR_SUPABASE_ANON_KEY"
      ]
    }
  ]
}
```

## Environment Switching

In `lib/environments.dart`, the `ENV` variable determines which configuration is used:

```dart
// dev environment uses dev Firebase options, dev Supabase URL, etc.
// prod environment uses prod Firebase options, prod Supabase URL, etc.
```

The `env.map()` / `env.when()` pattern is used throughout the codebase to switch behavior based on environment.

## Firebase Configuration

Firebase options are in separate files per environment:
- `lib/firebase_options_dev.dart` — Dev environment
- `lib/firebase_options.dart` — Production environment

These are generated by `flutterfire configure` and are NOT committed to git.
