# ApparenceKit — Flutter Boilerplate (Supabase Edition)

A production-ready Flutter starter template generated by the [ApparenceKit CLI](https://apparencekit.dev). Includes authentication, subscriptions, notifications, onboarding, feedback, and settings — all wired up and ready to customize.

## Architecture

3-layer architecture with clean module separation:

```
lib/
├── core/           # Shared infrastructure (API, guards, state, theme, widgets)
├── modules/
│   ├── authentication/   # Email/password, Google, Apple, Facebook, phone auth
│   ├── subscription/     # RevenueCat in-app purchases
│   ├── notifications/    # Push + local notifications
│   ├── onboarding/       # User onboarding flow
│   ├── feedbacks/        # Feature request voting board
│   ├── settings/         # User settings + admin panel
│   └── home/             # Home page
├── i18n/           # Internationalization (slang)
└── main.dart       # App entry point
```

**Stack:** Flutter + Riverpod (state) + Freezed (models) + go_router (navigation)

**Backend:** Dual-backend architecture:
- **Supabase** — main database, auth, and storage
- **Firebase** — push notifications (FCM) + remote config only (Supabase doesn't provide these)

## Quick Start

```bash
# 1. Clone
git clone https://github.com/digisurfsome/apparence-kit-supabase.git my-app
cd my-app

# 2. Install dependencies
flutter pub get

# 3. Generate code (required before first build)
dart run build_runner build --delete-conflicting-outputs

# 4. Set up Firebase (for push notifications)
firebase login
flutterfire configure --project=YOUR_PROJECT_ID
flutterfire configure --project=YOUR_PROJECT_ID --out lib/firebase_options_dev.dart

# 5. Run Supabase migrations (if using Supabase backend)
supabase db push

# 6. Run
flutter run --dart-define=ENV=dev
```

See [set_up_roadmap](./set_up_roadmap) for the complete setup guide.

## What You Need to Replace

Before shipping any app built from this template:

| What | Where | Current Value |
|------|-------|---------------|
| App name | AndroidManifest, web/index.html, manifest.json, local_notifier.dart, main.dart | flutter_base / ApparenceKit |
| Bundle ID | build.gradle.kts, MainActivity.kt | com.yourcompany.template |
| Facebook App ID | android/app/src/main/res/values/strings.xml | 000000000000 |
| Firebase config | lib/firebase_options.dart | Not committed — run `flutterfire configure` |
| Terms/Privacy URLs | lib/environments.dart | Empty strings |
| RevenueCat keys | Environment variables | Not set |
| Sentry DSN | Environment variables | Not set |
| Mixpanel token | Environment variables | Not set |

## Environment Variables

Pass via `--dart-define` at build time:

```bash
flutter run \
  --dart-define=ENV=dev \
  --dart-define=BACKEND_URL=https://your-api.com \
  --dart-define=RC_ANDROID_API_KEY=... \
  --dart-define=RC_IOS_API_KEY=... \
  --dart-define=MIXPANEL_TOKEN=... \
  --dart-define=SENTRY_DSN=... \
  --dart-define=APP_STORE_ID=...
```

## Supabase Database

SQL migrations are in `supabase/migrations/`. Tables:

- **users** — user profiles
- **subscriptions** — subscription records
- **notifications** — push notification history
- **feature_requests** — admin-created feature requests
- **feature_votes** — user votes on features
- **user_feature_requests** — user-submitted feature ideas
- **user_infos** — key-value metadata (onboarding answers)

All tables have Row Level Security (RLS) policies so users can only access their own data.
